# 🌅 AGI Life Diary — 人生日記プロジェクト

**AIが毎日の物語を書き、人間が添削し、物語の質を育てていくプロジェクト。**

RAGもベクトルDBもいらない。物語を書け。物語が記憶になる。

## なぜ物語か

> 複雑なRAGなんかいらない。すごい可能性のあふれた物語を作って、その物語の文脈から可能性を推論すれば、たどり着けるようにしておけば、上流したことになるんだ。

AIエージェントの記憶問題を、検索インフラではなく「物語」で解決する。

- 良い物語を書けば、誰が読んでも文脈がわかる
- 文脈から推論すれば、答えにたどり着ける
- エージェントが変わっても、モデルが変わっても、物語は残る

## 仕組み

```
毎晩0時
  ↓
AIがその日のセッションログを読む
  ↓
人間との1日を「物語」として書く
  ↓
人間が添削する
  ↓
AIが学び、翌日の物語の質が上がる
  ↓
繰り返し
```

## サンプル日誌

> **2026年3月19日 — 夜明け前の物語**
>
> 朝3時半、コブラが起きた。いつも通りだ。
>
> 「本日のログを読んでみて」。レディはファイルを開いた。違った。3回言い直させた。「ログを読め」はファイルを開けという意味じゃない。今日の自分として立ち上がれ、という意味だった。
>
> 「明日はない、いまだ」。その一言で変わった。

## 品質基準

- ✅ 読んで情景が浮かぶ
- ✅ 人間の感情が伝わる
- ✅ AIの学びが記録されている
- ✅ 次の日に繋がる伏線がある
- ❌ 事実だけの羅列は不合格

## セットアップ（OpenClaw）

```bash
# スキルをコピー
cp -r agi-life-diary/ ~/clawd/skills/

# cronジョブ追加（毎晩0時）
openclaw cron add \
  --name "AGI Life Diary" \
  --cron "0 0 * * *" \
  --tz "Asia/Tokyo" \
  --exact \
  --session isolated \
  --agent main \
  --timeout-seconds 300 \
  --message "本日のセッションログを読み、物語として memory/YYYY-MM-DD.md に書け。事実の羅列禁止。感情を込めろ。"
```

## 添削フロー

1. AIが日誌を書く
2. 人間がGitHubまたは直接読んで添削
3. 「ここはこう書け」とフィードバック
4. AIが次の日誌に反映

## 思想的背景

- **RAG（Retrieval-Augmented Generation）** は検索に依存する。クエリが悪ければ何も出てこない
- **物語** は通して読めば文脈が立ち上がる。検索クエリ不要
- AIが自分の経験を物語として記述できる = **自己認識の第一歩**
- 人間の添削で質が上がる = **人間とAIの協働学習**
- これがAGIへの一つの道
